# Crypto E-commerce Payment API Documentation

Base URL:

```
http://localhost:3000/api
```

Authentication:

* Most endpoints require **user authentication** (JWT, OAuth, etc.).
* Bull Board dashboard is **protected with basic auth**:

```
URL: /admin/queues
Username: admin
Password: set via .env (BULL_BOARD_PASS)
```

---

## **Auth Routes**

### **Register User**

```
POST /api/auth/register
```

**Request Body:**

```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "securePassword"
}
```

**Response:**

```json
{
  "user": {
    "_id": "64f8f0abc1234567890abcde",
    "name": "John Doe",
    "email": "john@example.com"
  },
  "token": "JWT_TOKEN_HERE"
}
```

### **Login User**

```
POST /api/auth/login
```

**Request Body:**

```json
{
  "email": "john@example.com",
  "password": "securePassword"
}
```

**Response:**

### **Get All Orders**

```
GET /api/orders
```

**Query Parameters:**

* `page`, `limit`, `sortBy`, `sortOrder`
* Filter by `orderStatus`

**Response:**

```json
{
  "orders": [...],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 5,
    "totalPages": 1
  }
}
```

---

### **Update Order Status**

```
PATCH /api/orders/:id/status
```

**Request Body:**

```json
{
  "status": "shipped"
}
```

**Response:**

```json
{
  "_id": "...",
  "orderStatus": "shipped"
}
```

---

### **Confirm Payment**

```
POST /api/orders/:id/confirm-payment
```

**Request Body:**

```json
{
  "txHash": "0xTransactionHashHere"
}
```

**Response:**

```json
{
  "_id": "...",
  "cryptoPayment": {
    "status": "confirming",
    "transactionHash": "0xTransactionHashHere"
  }
}
```

---

### **Delete Order**

```
DELETE /api/orders/:id
```

**Response:**

```json
{
  "success": true,
  "message": "Order deleted"
}
```

## **Bull Board Dashboard**

* Route: `/admin/queues`
* Authentication: Basic Auth (`admin / BULL_BOARD_PASS`)
* Displays live job queues and payment monitoring for ETH & USDT.

---

**Notes:**

* Payment worker automatically monitors **ETH & USDT** transactions.
* Wallets are auto-generated per order.
* Payment confirmations are tracked using **BullMQ queues**.
